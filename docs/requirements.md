# 요건 정의 — IT 운영팀 일감 관리 앱 (MVP)

---

## 기능 요건

| ID | 이름 | 설명 | 우선순위 |
|----|------|------|---------|
| F-01 | 일감 CRUD | 제목·설명·카테고리·우선순위·마감일을 포함한 일감을 생성·조회·수정·삭제한다. 모든 팀원이 등록 가능하며, 삭제는 확인 다이얼로그를 거친다. | 필수 |
| F-02 | 상태 이동 | 일감 상태를 `todo → in_progress → review → done` 순서로 이동한다. 모든 팀원이 자신의 담당 일감 상태를 직접 변경할 수 있다. | 필수 |
| F-03 | 담당자 배정 | 일감 생성·수정 시 팀원 목록에서 담당자를 지정한다. 팀장은 기존 일감의 담당자를 재배정할 수 있다. | 필수 |
| F-04 | 다차원 필터 | 담당자·상태·카테고리·우선순위 필터를 조합해 일감 목록을 실시간으로 좁힌다. 앱 진입 시 담당자 필터는 기본값 "전체"로 시작한다. | 필수 |
| F-05 | 대시보드 통계 | 전체 일감 수·진행 중·완료율(%)·기한 초과 건수를 상단 카드로 표시한다. 수치는 현재 필터 상태와 무관하게 팀 전체 기준으로 집계한다. | 높음 |

---

## 비기능 요건

| 영역 | 요건 |
|------|------|
| **성능** | 일감 목록(최대 200건) 초기 렌더링 ≤ 1 초 (LCP 기준, 사내 Wi-Fi 환경). 필터 변경 후 목록 갱신 ≤ 200 ms. |
| **보안** | 사내 네트워크(VPN 포함) 접속 시에만 앱에 접근 가능. 로컬스토리지에 개인정보(이름 외) 저장 금지. |
| **가용성** | 업무 시간(평일 08:00–19:00) 기준 월간 가동률 ≥ 99 %. 장애 발생 시 복구 목표 시간(RTO) ≤ 2 시간. |
| **접근성** | WCAG 2.1 AA 기준 준수. 키보드만으로 일감 생성·상태 이동·필터 전환의 핵심 3개 흐름을 완료할 수 있어야 한다. |
