# 사용자 스토리

> 대상 페르소나: **헬프데스크 담당자** (기술 수준 하 · 목록 뷰·필터 위주)

---

## 1. 핵심 시나리오

### 출근부터 퇴근까지 — 헬프데스크 담당자의 하루

| 시각 | 행동 | 앱 사용 내용 |
|------|------|-------------|
| 09:00 | **출근 직후 — 미처리 일감 파악** | 앱을 열어 목록 뷰에서 담당자 필터를 "나"로, 상태 필터를 `todo`로 설정한다. 어제 퇴근 후 등록된 신규 지원 요청을 확인한다. |
| 09:10 | **카테고리 필터로 내 담당 범위 좁히기** | 카테고리 필터를 `bug` + `other`로 조합해 시스템 관리자 담당(`infra`, `feature`)을 제외한 내 일감만 표시한다. |
| 09:20 | **우선순위 순으로 오전 작업 순서 결정** | 우선순위 필터를 `urgent → high` 순으로 전환하며 가장 먼저 처리할 일감을 선택한다. |
| 09:30 | **작업 시작 — 상태 전환** | 첫 번째 일감을 클릭해 상세 화면을 열고 "다음 상태로 이동" 버튼으로 `todo → in_progress`로 변경한다. |
| 10:30 | **신규 지원 요청 접수 — 일감 등록** | 전화로 받은 사용자 문의를 "새 일감" 폼에 입력한다. 제목·설명·카테고리(`bug`)·우선순위(`medium`)·마감일을 지정하고 자신을 담당자로 배정 후 저장한다. |
| 11:00 | **오전 처리 건 완료 처리** | 해결한 지원 요청의 상태를 `in_progress → done`으로 이동해 처리 이력을 남긴다. |
| 13:00 | **점심 후 복귀 — 대기 중 요청 재확인** | 상태 필터를 `todo`로 돌려 점심 시간 동안 새로 들어온 요청이 있는지 확인한다. |
| 14:00 | **마감일 임박 일감 집중 처리** | 마감일 컬럼을 기준으로 목록을 정렬해 주황(임박)·빨강(초과) 표시 일감을 최우선 처리한다. |
| 15:30 | **긴급 이슈 발생 — 우선순위 상향** | 처리 중 예상보다 심각한 장애임이 확인되면 해당 일감의 우선순위를 `medium → urgent`로 수정하고 팀장에게 구두 보고한다. |
| 17:30 | **퇴근 전 — 미완료 일감 상태 정리** | 오늘 미처리된 일감의 상태를 `in_progress`로 유지하고, 설명란에 "내일 오전 이어서 처리 예정" 메모를 남긴 뒤 수정을 저장한다. |
| 18:00 | **퇴근** | 목록 뷰에서 `done` 필터로 오늘 처리 완료 건수를 최종 확인하고 앱을 닫는다. |
