# 사용자 페르소나 — IT 운영팀

## 페르소나 표

| 역할 | 기술 수준 | 주요 시나리오 |
|------|-----------|--------------|
| **IT 운영팀장** | **상** — 전체 운영 현황 파악, SLA·에스컬레이션 관리 | 1. 긴급 장애·SLA 임박 일감을 대시보드에서 확인 → 담당자 재배정 <br>2. 팀 전체 처리율(완료율·기한 초과) 집계로 주간 보고 준비 <br>3. 반복 발생 이슈를 신규 일감으로 등록하고 카테고리·우선순위 지정 |
| **시스템 관리자** | **중** — 서버·네트워크·인프라 작업 중심, Kanban 보드 위주 사용 | 1. 배정된 인프라 작업 확인 → 상태를 `in_progress`로 전환 <br>2. 정기 점검 완료 후 `review`로 이동해 팀장 승인 요청 <br>3. 장애 발생 시 우선순위를 `urgent`로 긴급 상향 |
| **헬프데스크 담당자** | **하** — 목록 뷰·필터 위주, 사용자 지원 요청 처리 전담 | 1. 내 담당(`bug`·`other` 카테고리) 일감만 필터링해 확인 <br>2. 신규 사용자 지원 요청을 폼으로 빠르게 등록 <br>3. 처리 완료 건을 `done`으로 이동해 이력 관리 |

---

## MVP 헌장

> **범위 원칙**: 단일 테이블, 기능 5개 이하, 1일 안에 코드로 옮길 수 있는 분량.

위 3개 페르소나의 시나리오를 전부 커버하는 최소 기능 집합:

| # | 기능 | 관련 시나리오 |
|---|------|--------------|
| 1 | **일감 CRUD** — 생성·조회·수정·삭제 | 팀장 ③, 헬프데스크 ② |
| 2 | **상태 이동** — `todo → in_progress → review → done` | 시스템 관리자 ①②, 헬프데스크 ③ |
| 3 | **담당자 배정** — 팀원 선택 및 아바타 표시 | 팀장 ① |
| 4 | **우선순위·카테고리 필터** — 다차원 필터링 | 팀장 ①, 시스템 관리자 ③, 헬프데스크 ① |
| 5 | **대시보드 통계** — 전체·진행 중·완료율·기한 초과 4개 지표 | 팀장 ② |

MVP 범위 밖 (이후 이터레이션): 알림·에스컬레이션 자동화, 온콜 스케줄, SLA 규칙 엔진, 외부 티켓 시스템 연동.
